# Day 13 - Perform A Full New-User Dry Run

Why this day matters in the full project picture:
- a launch-ready repo is not real until it is tested end to end from a new-user perspective

Exact goal:
- perform one full new-user dry run on the resulting first-read and proof surfaces

Unlock condition:
- the launch-prep state survives a fresh-reader walkthrough without hidden assumptions

In scope:
- first-read walkthrough
- install/proof walkthrough
- note-taking on friction and contradictions

Out of scope:
- major feature work
- launch execution itself

Files to inspect first:
- README
- quickstart surface
- proof-pack surface
- launch texts

Expected deliverable:
- one dry-run result with concrete friction notes

Required verification:
- the real commands the dry run asks a user to perform

Key risks:
- testing from too much internal knowledge
- quietly compensating for missing instructions

Truth constraints:
- the dry run must use the repo as presented, not hidden operator context

Product constraints:
- keep the dry run focused on the flagship workflow

Success condition:
- remaining pre-launch gaps are explicit and small enough to fix or consciously defer
