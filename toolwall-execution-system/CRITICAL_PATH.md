1. Day 1 locks the role of `scripts/pack-smoke.mjs`
2. Day 2 locks the real checked-out boundary
3. Day 3 confirms the current serial proof surface
4. Day 4 sharpens README using verified truth
5. Day 5 writes the shortest safe quickstart off that README truth
6. Day 6 makes limits explicit so Day 7-12 cannot overclaim
7. Day 7 creates one durable demo path
8. Day 8 truth-syncs first-read surfaces so README/quickstart/limits/demo align
9. Day 9 cleans naming surfaces without breaking install/package truth
10. Day 10 assembles the proof pack from the now-aligned surfaces
11. Day 11 improves GitHub discovery around the same proof pack
12. Day 12 prepares launch texts from the verified discovery/proof surface
13. Day 13 runs a full new-user dry run against the launch-prep state
14. Day 14 executes the first launch wave

## Must Be Done Before Launch

- Days 1-13
- explicit proof/local-only truth separation
- one coherent first-read surface
- one coherent verification/proof surface
- one full new-user dry run
