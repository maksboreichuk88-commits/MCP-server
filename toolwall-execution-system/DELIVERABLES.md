# Deliverables

By Day 14, the expected final deliverables are:

- one coherent runtime boundary
- one verified local proof surface
- one README
- one quickstart
- one limits/non-goals doc
- one demo
- one launch-ready proof pack
- one GitHub-ready surface
- one set of public launch texts
- one dry-run result
- one first launch wave

Interpretation rules:
- "GitHub-ready surface" means a coherent repo/discovery surface, not automatic merge proof
- "public launch texts" means prepared and evidence-shaped launch copy, not proof that launch already happened
- "verified local proof surface" means the current serial local verification chain for the checked-out boundary, not public release proof
